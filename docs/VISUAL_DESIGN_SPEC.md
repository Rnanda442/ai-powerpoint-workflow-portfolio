# Portfolio Visual Design Specification

## Shared Rules

- Use one dominant idea per poster.
- Put the headline in the upper-left, 30-36 px, maximum six words.
- Use 18-22 px for stage labels and 14-16 px for supporting text.
- Keep important text inside a 40 px safe margin.
- Use one accent color for the main transformation and gray for supporting structure.
- Replace long sentences inside visuals with nouns and verbs.
- Give the final output 30-40% more visual weight than the input.
- Use motion only to explain a process: pulse, flow, reveal, classification, or uncertainty.
- On mobile, stack stages vertically or reduce them to three steps.

## 1. AI Agent Training

Current issue: screen, rubric, and agent have nearly equal weight.

Changes:

- Headline: `Screens Become Training Data`.
- Enlarge the future agent by 25%.
- Add a small red failure example beside the green rubric checks.
- Animate one cursor path from screen to rubric.
- Replace `human review` with `expert review`.
- Use the orange dot as a moving action trace, not decoration.

## 2. Critical-Mineral Knowledge Graph

Current issue: the graph and geology sketch compete for attention.

Changes:

- Headline: `From Thesis Notes To Graph`.
- Make the graph the dominant right-side object.
- Use three node colors only: deposit, mineral, evidence.
- Label one relationship as an example: `hosted in`.
- Reduce the prompt card text to three lines.
- Animate nodes entering the graph from the sketch.

## 3. Processing Earthquake Globe

Current issue: strong concept, but the poster does not show time or depth clearly.

Changes:

- Headline: `Earthquakes You Can See And Hear`.
- Add a depth legend from shallow yellow to deep red.
- Add a thin orbit/time arc around the globe.
- Keep pulsing epicenters, but vary pulse speed by magnitude.
- Make the sound waves react outward from the largest event.
- Replace the code box with three chips: `location`, `depth`, `magnitude`.

## 4. Seismic Processing

Current issue: the pick and the output map feel disconnected.

Changes:

- Headline: `Waveform To Defensible Pick`.
- Enlarge the P-pick line and label it directly.
- Add a confidence band around the pick.
- Use one arrow labeled `QA`.
- Replace generic output curves with a small station/event map.
- Animate the pick line settling after two candidate positions.

## 5. North Slope Energy Screening

Current issue: the poster is flatter and less convincing than the real Structural Explorer.

Changes:

- Headline: `Public Data To 3D Screening`.
- Use a perspective structural stack based on the real four horizons.
- Show wells piercing the surfaces.
- Add assessment-unit outlines as a top overlay.
- Use the actual explorer colors consistently.
- End with one highlighted target zone, clearly labeled `screen, not prediction`.
- Link directly to the interactive Structural Explorer.

## 6. Rock Resource Mapping

Current issue: the range bars and 3D block are visually disconnected.

Changes:

- Headline: `Signals Become Rock Classes`.
- Connect each input range to a matching colored zone in the block.
- Add a small legend for gravity, EM, and geochemistry.
- Make the output block 35% larger.
- Add uncertainty hatching where classifications overlap.
- Animate layers filling as thresholds are applied.

## 7. Field Geophysics

Current issue: three methods are shown, but their combination is not.

Changes:

- Headline: `Different Methods, One Subsurface`.
- Add a fourth combined panel after gravity, EM/ERT, and seismic.
- Put uncertainty bars under each method.
- Replace the large question mark with a transparent overlap area.
- Use consistent subsurface geometry across all panels.
- Animate each method fading into the combined interpretation.

## 8. Moho ML Transfer

Current issue: the model diagram is readable, but transfer risk is understated.

Changes:

- Headline: `Train Here. Test Somewhere Else.`
- Increase the Australia and USA labels to 24 px.
- Add a visible train/test boundary.
- Make the residual map the final dominant element.
- Label leakage risk with a red gate before transfer.
- Animate failed predictions appearing as red residuals.

## 9. Codex App Pipeline

Current issue: the workflow ends at GitHub/app, while model validation looks optional.

Changes:

- Headline: `Files Become A Tested App`.
- Put the leakage check directly in the main path.
- Add Streamlit as a distinct stage after GitHub.
- Use a loop arrow from test results back to Codex.
- Replace `downloads` with `existing work`.
- Animate files consolidating into one tracked repository.

## Typography And Layout

- Page title: 38-44 px desktop, 30-34 px mobile.
- Section title: 26-30 px desktop, 22-26 px mobile.
- Poster headline: 30-36 px.
- Poster stage label: 18-22 px.
- Poster detail: 14-16 px.
- Body copy: keep paragraphs under 55 words.
- Use bold only for the claim, project name, or decision.
- Put technical caveats in expanders rather than the default reading path.
